# 市场 T 度划分说明

本系统将国家/地区按**市场 T 度**划分为四类：**亚洲 T1**、**欧美 T1**、**T2**、**T3**，用于汇总分国家数据（如获量、收入）为各 T 度市场的汇总指标。划分规则以本节表格为准。

## 一、T 度定义（按国家/地区）

| 市场 T 度 | 描述 | 包含地区（部分） |
|-----------|------|------------------|
| **亚洲 T1** | 高价值核心亚洲市场（**排除 CN**），付费能力强、CPI 高、买量与发行重点区域 | TW（台湾）、HK（香港）、JP（日本）、KR（韩国） |
| **欧美 T1** | 高价值核心欧美市场，ARPU 最高、投放竞争激烈、产品验证主战场 | US（美国）、GB（英国）、DE（德国）、FR（法国）、CA（加拿大）、AU（澳大利亚） |
| **T2** | 中高价值成熟市场，具备稳定付费能力，常作为第二阶段扩量市场 | IT（意大利）、ES（西班牙）、NL（荷兰）、SE（瑞典）、NO（挪威）、DK（丹麦）、CH（瑞士）、AT（奥地利）、BE（比利时）、NZ（新西兰）、SG（新加坡）、IL（以色列）、AE（阿联酋）、SA（沙特） |
| **T3** | 高量级新兴市场，CPI 低、装量大，常用于规模冲量与长尾覆盖 | BR（巴西）、MX（墨西哥）、AR（阿根廷）、CO（哥伦比亚）、CL（智利）、PE（秘鲁）、RU（俄罗斯）、TR（土耳其）、PL（波兰）、RO（罗马尼亚）、UA（乌克兰）、IN（印度）、ID（印尼）、TH（泰国）、VN（越南）、PH（菲律宾）、MY（马来西亚）、NG（尼日利亚）、ZA（南非）、IQ（伊拉克）等 |

**重要规则**：**表内未出现的国家代码**，或 **API 请求到的地区数据中不在上表（亚洲 T1 / 欧美 T1 / T2）列表中的国家代码**，一律归入 **T3**。

## 二、在系统中的使用

- **分地区数据**：`request/fetch_country_data.py` 拉取的是**国家维度**数据（如 `request/country_data/json/{app_id}.json`），每条记录含 `country`（国家代码）、`unified_units`、`unified_revenue` 等。
- **按 T 度汇总**：将国家维度数据按 **mapping/市场T度.csv** 映射到亚洲 T1、欧美 T1、T2、T3；**未在映射表中的国家代码默认归为 T3**，再汇总得到各 T 度的获量/收入（如「亚洲T1_安装」「欧美T1_流水」等列）。
- **final_join 表**：存放的是**目标产品表 + 各 T 度市场获量**（无广告列）。即：目标产品基础列 + 亚洲 T1 / 欧美 T1 / T2 / T3 的安装、流水等汇总列，供前端或分析使用。
- **广告拉取**：`request/fetch_ad_creatives.py` 按**地区标签**（亚洲 T1、欧美 T1、T2、T3）请求广告创意时，与上述 T 度划分一致。

## 三、映射表维护

- 文件：**mapping/市场T度.csv**
- 列：`country`（国家代码）、`T度`（取值：亚洲T1、欧美T1、T2）
- 仅需列出**亚洲 T1、欧美 T1、T2** 的国家；脚本在汇总时，**未出现在该表中的国家代码一律归为 T3**。
