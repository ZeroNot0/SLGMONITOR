# 前端技术选型说明

之前前端是 **CSS + JS + HTML** 静态网页，代码丢失后需要恢复。当前可选方案对比如下。

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **静态单页（HTML + CSS + JS）** | 无构建、无 node、易备份、易部署、可直接用 `start_server.py` 或任意静态服务器 | 逻辑多了以后难维护、无组件化 | 快速恢复、内部看板、功能以表格+筛选为主 |
| **Vue 3 + Vite** | 组件化、响应式、单文件组件、生态成熟、构建快 | 需要 node/npm、需 build 后部署 | 后续要加多页、产品/素材维度、状态较多时 |
| **React + Vite** | 生态大、组件化、类型可选 TypeScript | 需要 node/npm、需 build | 同上，团队更熟 React 时 |
| **Flask/FastAPI + 模板** | 服务端渲染、易做登录/权限、SEO 友好 | 需要 Python 后端常驻、前后端耦合 | 需要登录、权限、服务端逻辑时 |

## 推荐路径

1. **先恢复静态单页**（当前已采用）  
   - 一个 `index.html` + `css/style.css` + `js/app.js`，实现：顶部导航、左侧年/周选择、主内容区数据表（带格式、搜索、排序）。  
   - 不依赖 node，直接放在 `frontend/` 下，用项目里的 `start_server.py` 提供静态资源即可。  
   - 数据仍读**带格式的 JSON**（`frontend/data/{年}/{周}_formatted.json`），与现有 `convert_excel_with_format.py` 流程一致。

2. **后续如需扩展再考虑框架**  
   - 若增加「产品维度 / 素材维度 / 组合分析」等多视图、状态较多，再迁到 **Vue 3 + Vite** 或 **React + Vite**。  
   - 数据接口可保持不变（仍读静态 JSON 或由同一 `start_server.py` 提供简单 API），仅前端改为 SPA。

## 当前前端形态

- **格式**：静态单页（HTML + CSS + JS），无构建工具。  
- **入口**：`frontend/index.html`，通过 `python start_server.py` 访问 `http://localhost:8000/frontend/`。  
- **数据**：优先加载 `frontend/data/{year}/{week_tag}_formatted.json`；若需支持直接读 Excel，可后续加（需引入如 SheetJS 等库）。
